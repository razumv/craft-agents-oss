## Агент 1: Trend Scout (Анализ трендов)

### Цель
Ежедневно собирать и ранжировать топ-10 тем из крипто-ниши и глобальных новостей, чтобы маркетинг-команда утром получала готовый список идей для контента.

### Источники данных

| Источник | Метод сбора | Сложность | Риск бана |
|----------|------------|-----------|-----------|
| Twitter/X | API v2 (платный) или Nitter-парсинг | Средняя | Низкий (API) / Средний (парсинг) |
| Reddit | Reddit API (бесплатный tier) | Низкая | Низкий |
| Cointelegraph | RSS + парсинг | Низкая | Минимальный |
| CoinDesk | RSS + парсинг | Низкая | Минимальный |
| Decrypt | RSS | Низкая | Минимальный |
| Google Trends | Unofficial API / SerpAPI | Средняя | Низкий |
| YouTube Trending | YouTube Data API | Низкая | Низкий |

### Логика работы

```
1. [06:00] Cron запускает сбор данных из всех источников
2. Агрегация: объединение по темам (NLP-кластеризация)
3. Ранжирование по:
   - Частота упоминаний (вес: 30%)
   - Скорость роста (вес: 25%)
   - Вовлеченность (лайки/RT/комменты) (вес: 25%)
   - Релевантность к крипто/DoubleTOP (вес: 20%)
4. LLM-обработка: краткое описание каждой темы + почему она актуальна
5. [08:00] Отправка в Telegram/чат: "Топ-10 тем на сегодня"
6. [12:00] Обновление: новые темы за первую половину дня
```

### Выход (output)

```json
{
  "date": "2026-03-17",
  "topics": [
    {
      "rank": 1,
      "title": "SEC одобрила новый ETF на Ethereum стейкинг",
      "summary": "Комиссия по ценным бумагам...",
      "sources": ["twitter:432", "reddit:89", "cointelegraph:2"],
      "momentum": "rising",
      "relevance_score": 0.92,
      "suggested_angles": [
        "Обзор: что это значит для рынка",
        "Сравнение с BTC ETF",
        "Как это повлияет на DeFi"
      ]
    }
  ]
}
```

### Варианты реализации

#### Вариант A: Python-сервис + Cron
- **Стек:** Python, FastAPI, Celery + Redis, PostgreSQL
- **LLM:** OpenAI GPT-4o / Claude через API
- **Плюсы:** Полный контроль, гибкость, низкая стоимость
- **Минусы:** Больше кода, нужен DevOps
- **Оценка:** 2-3 недели на MVP

#### Вариант B: n8n / Make.com workflow
- **Стек:** n8n (self-hosted) + LLM node
- **Плюсы:** Быстрая настройка, визуальный workflow, легко менять
- **Минусы:** Ограничения при масштабировании, зависимость от платформы
- **Оценка:** 1 неделя на MVP

#### Вариант C: LangChain/CrewAI агент
- **Стек:** Python, LangChain/CrewAI, встроенные tools
- **Плюсы:** Продвинутая логика, цепочки рассуждений, легко добавлять tools
- **Минусы:** Overhead на фреймворк, дороже по токенам
- **Оценка:** 2 недели на MVP

**Рекомендация:** Вариант A для долгосрочной стабильности, Вариант B для быстрого старта и тестирования гипотезы.

## Агент 2: Content Scriptwriter (Сценарист)

### Цель
По выбранной теме генерировать готовый сценарий для видео/поста/reels с учетом тона бренда, формата и целевой аудитории.

### Входные данные
- Тема (от Trend Scout или вручную)
- Формат: Short (< 60 сек) / Long (5-15 мин) / Пост / Thread / Статья
- Тон: Образовательный / Новостной / Развлекательный / Экспертный
- Референсы (опционально): ссылки на похожий контент

### Логика работы

```
1. Получение темы + параметров
2. Поиск контекста:
   - Свежие данные по теме (из Trend Scout)
   - Позиция DoubleTOP по теме (из базы знаний бренда)
   - Успешные примеры контента конкурентов по теме
3. Генерация сценария:
   - Hook (первые 3 сек для привлечения)
   - Основная часть (структурированная по блокам)
   - CTA (призыв к действию)
4. Адаптация под формат:
   - Short: тайминг по секундам, визуальные указания
   - Long: разбивка на главы, B-roll подсказки
   - Пост: заголовок + тело + хэштеги
5. Вывод на ревью оператору
```

### Выход (output) — пример для Short

```
HOOK (0-3 сек): "SEC только что изменила правила игры для Ethereum"
СЦЕНА 1 (3-12 сек): [Говорящая голова] Объяснение что произошло
СЦЕНА 2 (12-25 сек): [Графика] Сравнение с предыдущими ETF
СЦЕНА 3 (25-40 сек): [Говорящая голова] Что это значит для инвесторов
CTA (40-50 сек): "Подписывайтесь, чтобы не пропустить следующий инсайт"

Хэштеги: #ethereum #ETF #crypto #SEC #DeFi
Время публикации: 14:00-16:00 UTC (пик активности)
```

### Варианты реализации

#### Вариант A: Prompt-chain с brand knowledge base
- **Стек:** Python, LLM API, Vector DB (для базы знаний бренда)
- **Плюсы:** Глубокая кастомизация, бренд-консистентность
- **Минусы:** Нужно собрать и поддерживать базу знаний
- **Оценка:** 2 недели на MVP

#### Вариант B: Fine-tuned модель на контенте DoubleTOP
- **Стек:** Fine-tune GPT-4o / Claude на существующем контенте
- **Плюсы:** Лучшее попадание в тон, меньше промптинга
- **Минусы:** Дороже, нужны данные для обучения (50+ примеров)
- **Оценка:** 3-4 недели (включая подготовку данных)

#### Вариант C: Template-based + LLM fill
- **Стек:** Набор шаблонов по форматам + LLM заполняет слоты
- **Плюсы:** Быстро, предсказуемо, дешево
- **Минусы:** Менее креативно, может быть шаблонно
- **Оценка:** 1 неделя на MVP

**Рекомендация:** Вариант A — оптимальный баланс. Вариант C можно использовать как быстрый старт, потом мигрировать на A.

## Агент 3: Social Analytics (Аналитика соцсетей)

### Цель
Автоматический ежедневный сбор метрик по всем соцсетям, отслеживание динамики и корреляции между постами и результатами.

### Метрики по платформам

| Платформа | Метрики | Метод сбора |
|-----------|---------|-------------|
| Instagram | Подписчики, охват, вовлеченность, сохранения, shares, stories views | Instagram Graph API (Business) |
| Telegram | Подписчики, просмотры постов, репосты, реакции | Telegram Bot API + TGStat API |
| YouTube | Подписчики, просмотры, watch time, CTR превью, комментарии | YouTube Data API + YouTube Analytics API |
| Twitter/X | Подписчики, показы, вовлеченность, клики, RT | Twitter API v2 |

### Логика работы

```
1. [00:30] Ежедневный сбор метрик по всем платформам
2. Сохранение в БД с привязкой к дате
3. Расчет:
   - Дневная/недельная/месячная динамика
   - Корреляция: какой пост → какой всплеск
   - Бенчмарк: сравнение с средними показателями
4. [09:00] Ежедневный дайджест в Telegram:
   - Ключевые метрики за вчера
   - Топ-3 поста по вовлеченности
   - Аномалии (резкий рост/падение)
5. Еженедельный отчет (понедельник):
   - Графики трендов
   - Рекомендации по контенту
```

### Выход — ежедневный дайджест

```
📊 Дайджест 17.03.2026

Instagram: 45,230 (+120) | Охват: 12.5K | ER: 4.2%
Telegram:  23,100 (+85)  | Views: 8.7K  | Shares: 340
YouTube:   18,400 (+45)  | Views: 5.2K  | Watch: 2.1hr avg
Twitter:   12,800 (+200) | Impr: 45K    | Eng: 3.8%

🔥 Топ-пост: Instagram Reel "Ethereum ETF" — 45K views, 8.2% ER
⚠️ Аномалия: Twitter +200 подписчиков (обычно +50) — проверить причину
```

### Варианты реализации

#### Вариант A: Собственный сервис с API-интеграциями
- **Стек:** Python, FastAPI, PostgreSQL, Grafana для визуализации
- **Плюсы:** Полный контроль, кастомные метрики, дешево
- **Минусы:** Нужно поддерживать API-интеграции
- **Оценка:** 3 недели на MVP

#### Вариант B: SocialBlade/HypeAuditor API + агрегатор
- **Стек:** API третьих сервисов + свой агрегатор
- **Плюсы:** Меньше кода, данные уже обработаны
- **Минусы:** Платные API, ограничения по данным
- **Оценка:** 1-2 недели на MVP

#### Вариант C: Google Sheets + Apps Script + LLM
- **Стек:** Google Sheets как "БД", Apps Script для сбора, LLM для анализа
- **Плюсы:** Быстро, понятно команде, не нужен DevOps
- **Минусы:** Не масштабируется, ограничения Sheets
- **Оценка:** 1 неделя на MVP

**Рекомендация:** Вариант A для production. Для быстрого старта — Вариант C (команда сразу начнет получать данные, потом мигрируем).

## Агент 4: Ad Analytics (Аналитика рекламы Meta)

### Цель
Ежедневные автоматические отчеты по рекламным кампаниям Meta с ключевыми метриками и алертами.

### Метрики

| Метрика | Описание |
|---------|----------|
| CPF (Cost per Follow) | Цена за подписку |
| CPM | Цена за 1000 показов |
| CTR | Click-through rate |
| CPC | Cost per click |
| Reach | Уникальный охват |
| Frequency | Средняя частота показа |
| ROAS | Return on Ad Spend (если есть e-commerce) |
| Spend | Расход за день |

### Логика работы

```
1. [07:00] Запрос данных через Meta Marketing API
   ⚠️ КРИТИЧНО: Использовать ТОЛЬКО официальный API
   ⚠️ НЕ подключать сторонние инструменты к кабинету
2. Агрегация по:
   - Кампаниям
   - Адсетам
   - Креативам
3. Сравнение с:
   - Вчерашним днем
   - Средним за 7 дней
   - Целевыми KPI
4. Алерты:
   - CPF выше порога → уведомление
   - Бюджет израсходован > 80% → уведомление
   - CTR упал > 20% → уведомление
5. [09:30] Отправка отчета в Telegram
```

### Безопасность (ПРИОРИТЕТ)

```
✅ РАЗРЕШЕНО:
- Meta Marketing API (официальный, через App Review)
- Facebook Business SDK
- Собственный сервер с OAuth токеном

❌ ЗАПРЕЩЕНО:
- Manus/Cloud напрямую к кабинету
- Браузерная автоматизация (Puppeteer/Playwright) на Meta
- Сторонние SaaS с доступом к кабинету
- Скрейпинг Meta Ads Manager UI
```

### Варианты реализации

#### Вариант A: Meta Marketing API + собственный сервис
- **Стек:** Python, Meta Business SDK, PostgreSQL
- **Плюсы:** Безопасно, официально, полный контроль
- **Минусы:** Нужен App Review от Meta (1-2 недели), настройка OAuth
- **Оценка:** 2-3 недели (с App Review)

#### Вариант B: Supermetrics / Funnel.io + агрегатор
- **Стек:** Платный коннектор + свой анализатор
- **Плюсы:** Быстро, надежно, не нужен App Review
- **Минусы:** Платная подписка ($50-200/мес)
- **Оценка:** 1 неделя

#### Вариант C: Google Sheets + Meta API (через Sheets add-on)
- **Стек:** Supermetrics for Sheets / JEPTO
- **Плюсы:** Данные в привычном формате, быстро
- **Минусы:** Ограничения, платный add-on
- **Оценка:** 3-5 дней

**Рекомендация:** Вариант A для долгосрочной безопасности. Вариант B как быстрый старт (если бюджет позволяет).

## Агент 5: Comment Manager (Менеджер комментариев)

### Цель
Парсить комментарии со всех платформ, оценивать sentiment, выявлять популярные вопросы и генерировать идеи для нового контента.

### Источники

| Источник | Метод | Частота |
|----------|-------|---------|
| Instagram комментарии | Graph API | Каждые 4 часа |
| Twitter реплаи/mentions | Twitter API v2 | Каждые 2 часа |
| Telegram комментарии | Bot API (в группах с комментариями) | Real-time (webhook) |
| YouTube комментарии | YouTube Data API | Каждые 6 часов |
| Instagram DM (Антон/Гнат) | Требует исследования* | По запросу |
| Squad Education обсуждения | Зависит от платформы | Ежедневно |

*Парсинг DM — серая зона, нужно исследовать легальные методы

### Логика работы

```
1. Сбор комментариев по расписанию
2. NLP-анализ:
   - Sentiment: позитивный / нейтральный / негативный
   - Категоризация: вопрос / отзыв / жалоба / предложение / спам
   - Извлечение ключевых тем
3. Кластеризация вопросов:
   - "Как начать?" → образовательный контент
   - "Когда обновление?" → продуктовый контент
   - "Проблема с X" → тех. поддержка / обратная связь продукту
4. Генерация тем:
   - Топ-5 вопросов за неделю → идеи для видео/постов
   - Негативные тренды → алерты команде
5. Еженедельный отчет:
   - Общий sentiment
   - Популярные темы из комментариев
   - Рекомендации для контент-плана
```

### Выход — еженедельный отчет

```
📝 Комментарии: неделя 11-17 марта

Всего: 1,240 | 😊 68% | 😐 22% | 😡 10%

Топ-5 вопросов:
1. "Как работает стейкинг?" (87 раз) → РЕКОМЕНДАЦИЯ: обучающее видео
2. "Когда листинг на Binance?" (54 раз) → ПЕРЕДАТЬ ПРОДУКТУ
3. "Безопасно ли хранить на платформе?" (41 раз) → РЕКОМЕНДАЦИЯ: видео о безопасности
4. "Какие комиссии?" (38 раз) → РЕКОМЕНДАЦИЯ: сравнительный пост
5. "Как вывести средства?" (29 раз) → РЕКОМЕНДАЦИЯ: туториал

⚠️ Негативный тренд: жалобы на скорость вывода (+30% за неделю)
```

### Варианты реализации

#### Вариант A: Собственный NLP-pipeline
- **Стек:** Python, spaCy/transformers для NLP, LLM для генерации идей
- **Плюсы:** Полный контроль, кастомизация под крипто-термины
- **Минусы:** Сложнее в разработке
- **Оценка:** 3-4 недели

#### Вариант B: LLM-first подход
- **Стек:** Python, GPT-4o/Claude для всего анализа
- **Плюсы:** Быстро, высокое качество анализа, гибко
- **Минусы:** Дороже по токенам при больших объемах
- **Оценка:** 2 недели

#### Вариант C: Brand24 / Mention + LLM-обработка
- **Стек:** Платный сервис для сбора + LLM для анализа
- **Плюсы:** Сбор уже решен, фокус на аналитику
- **Минусы:** Платный ($100-300/мес), может не покрывать все источники
- **Оценка:** 1-2 недели

**Рекомендация:** Вариант B — лучший баланс скорости и качества для MVP. Потом оптимизируем дорогие LLM-вызовы классическим NLP.

## Агент 6: Video Processor (Обработка видео)

### Цель
Анализ существующих видео конкурентов, разбивка на сцены, генерация аналогичного контента с сохранением структуры сценария.

### Логика работы

```
1. Вход: ссылка на YouTube/Shorts
2. Скачивание + транскрипция (Whisper)
3. Разбивка на сцены:
   - По визуальным переходам (OpenCV / scene detection)
   - По смысловым блокам (из транскрипта)
   - Тайминг ~8 сек на сцену
4. Анализ сценария:
   - Hook / Body / CTA
   - Тон и стиль
   - Визуальные приемы
5. Генерация нового сценария:
   - Сохранение структуры
   - Адаптация под бренд DoubleTOP
   - Новые визуальные указания
```

### Варианты реализации

#### Вариант A: Python pipeline (yt-dlp + Whisper + OpenCV + LLM)
- **Стек:** yt-dlp, OpenAI Whisper, PySceneDetect, LLM
- **Плюсы:** Полный контроль, open-source
- **Минусы:** GPU для Whisper, сложность pipeline
- **Оценка:** 3-4 недели

#### Вариант B: Облачные API (AssemblyAI + LLM)
- **Стек:** AssemblyAI (транскрипция + scene), LLM
- **Плюсы:** Не нужен GPU, высокое качество
- **Минусы:** Платно по минутам
- **Оценка:** 2 недели

**Рекомендация:** Вариант B для старта (быстрее, не нужен GPU). Вариант A если объемы вырастут и нужно экономить.

## Агент 7: Article Writer (Автор статей)

### Цель
Мониторинг конкурентов, создание статей и органическое участие в Twitter-сообществе.

### Логика работы

```
1. Мониторинг конкурентов:
   - Блоги (RSS)
   - Twitter-аккаунты
   - Medium / Mirror публикации
2. Анализ:
   - О чем пишут конкуренты?
   - Что у DoubleTOP лучше/хуже?
   - Где есть неосвещенные ниши?
3. Генерация статей:
   - SEO-оптимизация (keywords, meta)
   - Тон: экспертный, не промо
   - Длина: 800-2000 слов
4. Twitter-активность:
   ⚠️ ВАЖНО: общение как участник сообщества
   - Ответы на релевантные треды
   - Собственные мнения по темам
   - БЕЗ ссылок на DoubleTOP
   - Цель: органический рост через экспертизу
```

### Варианты реализации

#### Вариант A: LLM + RSS monitoring + SEO tools
- **Стек:** Python, feedparser, LLM, SEO API (Ahrefs/SEMrush)
- **Плюсы:** Полноценный pipeline, SEO-оптимизация
- **Минусы:** Больше интеграций
- **Оценка:** 2-3 недели

#### Вариант B: CrewAI multi-agent
- **Стек:** CrewAI с ролями (Researcher, Writer, Editor, SEO)
- **Плюсы:** Разделение ответственности, лучшее качество
- **Минусы:** Дороже по токенам, сложнее отладка
- **Оценка:** 2-3 недели

**Рекомендация:** Вариант A для MVP. Вариант B если качество статей окажется недостаточным.

### Phase 1: Trend Scout MVP (Недели 1-3)
| Задача | Стек | Результат |
|--------|------|-----------|
| Trend Scout — сбор данных | Python, feedparser, Reddit API, Twitter API | Парсинг 7 источников |

| Trend Scout — доставка | Telegram Bot API | Ежедневный список тем в 08:00 и 12:00 |
| Craft Automation | SchedulerTick cron | Автозапуск по расписанию |

### Phase 2: Content + Meta Analytics (Недели 3-6)
| Задача | Стек | Результат |
|--------|------|-----------|
| Content Scriptwriter | Python, Claude Opus, Vector DB | Генерация сценариев по запросу |
| Ad Analytics (Meta) | Python, Meta Marketing API | Ежедневные отчёты по рекламе |
| Brand Knowledge Base | Embeddings + Vector DB | База тона/стиля DoubleTOP |

### Phase 3: Full Analytics Suite (Недели 6-10)
| Задача | Стек | Результат |
|--------|------|-----------|
| Social Analytics | Python, Platform APIs, PostgreSQL | Ежедневная аналитика всех соцсетей |
| Comment Manager | Python, Claude Sonnet | Еженедельные инсайты из комментариев |
| Article Writer | Python, Claude Opus, SEO APIs | Статьи + Twitter-присутствие |
| Video Processor | Python, AssemblyAI, Claude Sonnet | Анализ и генерация видео-сценариев |

### Phase 4: Dashboard & Integrations (Недели 10-14)

- Запуск/остановка агентов из UI
- Исторические данные и графики
